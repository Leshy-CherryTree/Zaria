Basics

Zone files

All game data, except for data inherently binary in nature, such as audio, images and meshes, is stored in ZONE format. ZONE is a text format that is derived from JSON.

All Zone files can be edited in text mode, and most of them also in graph mode.

More on editing Zone Files can be found here.

Libraries

To organize data Zone Libraries are used. A library is a file holding paths to Zone files.

A library is needed to edit maps and to run a game based on the Zaria Framework.

More on editing Zone Libraries can be found here.

Asset directories

Game assets are located in 3 directories

  • Assets - Assets that do not need conversion are to be put there. This directory is later packed into a .jar file and used as a source of assets for the running game.
  • Work Files - Assets that need conversion are to be put here. After conversion their counterparts are automatically put in the Assets directory.
  • Scripts - Only for scripts. This directory is packaged into a single jar for use by the game.

More on Assets, Work Files and Conversion can be found here.
