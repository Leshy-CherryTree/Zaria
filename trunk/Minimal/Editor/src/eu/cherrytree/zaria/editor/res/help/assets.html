Assets and Work Files

Assets are mainly stored in two directories: Assets and Work Files. There is a clear distinction what kind of file should be where.

When editing a Zone file in the Assets directory a different set of Object Definitions will available compared to a Zone file from the Work Files directory.

Assets directory

Assets directory should contain files that are to be used directly by the game and don't need conversion:

  • Textures
  • Audio files
  • Models in .j3o format
  • Zone files that are used directly by the game (most of them).

This is also the directory used for storing assets by the jMonkey Engine SDK.

Work Files directory

Work Files should contain files that need to processed before they can be used by the game:

  • Meshes in .obj and .blend formats
  • Maps
  • Zone file containing definitions used for map creation.

Conversion

Right now files that need conversion are:

  • Meshes in .obj and .blend formats
  • Maps

To convert meshes double click on it in the file view. A converted version will be created in the Assets directory using the same relative path.

Meshes after conversion are stripped of their materials. To be used properly a mesh needs to be part of a GeometryDefinition.

Maps used unconverted meshes through MapGeometryDefinitions. If a mesh is only to be part of the map geometry it doesn't need to be converted.

In the file view Meshes are are shown using icon, if they have been already converted and if not. If the file format is no supported wil be used.
