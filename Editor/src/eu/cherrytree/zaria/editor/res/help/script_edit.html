Editing Scripts

Scripting in Zaria is done using JavaScript syntax with small additions. To learn more about JavaScript in general visit JavaScript section on MDN.

Imports

The most significant difference between normal JavaScript and ZoneScript is the addition of imports.

The syntax is the same as imports in Java:

    import path.to.file;

The path takes the Scripts directory as root. The .zonescript extension is not part of the import.

Imports work similarly to imports in Objective-C. All of the source file is put in place of the import command. However the preprocessor makes sure, that nothing is duplicated and no cycles exist. That means that you can use all variables and functions defined from the included file.

Keep in mind that every script has its own context, so changing a value of a variable from the imported file will not impact any other script, than the one which is importing.

You can drag script files from the file view on the right into the editing area. Proper syntax for importing the file will be generated automatically.

Function list

On the right you will have a list of functions that are available to the script. You can drag any of them from the list into the editing area.

Packaging

For the scripts to be usable in game they have to be packaged. To do that press . That will generate a scripts.jar file at the location specified in config.xml.
